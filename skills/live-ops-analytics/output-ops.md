# 运营自用复盘报告规范

## 风格定位
简洁、直接、口语化。运营同学自己用，不需要背景铺垫，直接说问题和下一步怎么做。

---

## 报告模板

```
# [场次/视频名] 复盘｜[日期]

## 本场一句话总结
[最重要的结论，一句话，例如："停留时长掉了 40%，主要是开场节奏太慢，下场要改"]

## 核心数据
| 指标 | 本场 | 上场 | 变化 |
|------|------|------|------|
| 场观 UV | | | ⚠️/✅ |
| 在线峰值 | | | |
| 人均停留 | | | |
| 互动率 | | | |
| 下单转化率 | | | |
| GMV | | | |
| ROI（如有投流） | | | |

异常值标 ⚠️，正常或提升标 ✅

## 做得好的（保持）
- [具体操作] → 带来了 [具体数据结果]

## 有问题的（下场改）
- ⚠️ [问题] → 可能原因：[XXX] → 下场怎么改：[具体动作]

## 下场行动清单
🔴 必改：[XXX]
🟡 试试：[XXX]
🟢 保持：[XXX]

## 下场目标参考
场观目标：XX | GMV 目标：XX | 转化率目标：XX%
```

---

## 注意事项
- 不需要写背景介绍，直接进数据
- 建议用缩写和运营黑话没关系（GPM、UV、CPM 直接用）
- 优先级标签用 🔴🟡🟢，视觉扫描快
- 弹幕分析结论直接说"用户最怕的是 XXX，下场话术要加 XXX"
