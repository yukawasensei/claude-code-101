## 二、核心分析模块

### 模块 A：回复覆盖率分析

```
总体回复覆盖率 = 已回复 / (已回复 + 未回复 + 无需回复) × 100%
有效回复率 = 已回复 / (已回复 + 未回复) × 100%  ← 更重要，排除表情包等无需回复
```

| 指标 | 健康基准 | 预警 |
|---|---|---|
| 有效回复率 | >85% | <70% 说明脚本覆盖不足 |
| 购买意向弹幕回复率 | >90% | <75% 说明在丢失转化机会 |
| 疑虑类弹幕回复率 | >85% | <70% 说明异议未被承接 |

---

### 模块 B：AI 误判分析（⚠️ 最重要的诊断项）

**操作步骤：**
1. 筛选 `回复状态 = 未回复` 的所有行
2. 查看 `补充说明` 列，分类统计原因：

| 补充说明类型 | 含义 | 处理建议 |
|---|---|---|
| AI检测与本直播间无关不作回复 | **AI误判** — 实际是相关提问 | ⚠️ 逐条人工核查，找出误判规律 |
| 弹幕内容为表情包，无需回复 | 正常过滤 | 无需处理 |
| AI检测为无意义弹幕，不回复 | 需抽样核查是否真无意义 | 抽查5-10条确认 |
| 意图规则频率限制，无需回复 | 触发频率限制 | 评估限制是否过严 |

3. 对"AI检测与本直播间无关"的弹幕逐条看内容，按类型归类：

```
【AI 误判弹幕归类】
- 尺码/身材咨询类：X条（如"170/50kg穿什么尺码"）
- 商品现货/库存类：X条
- 商品特写/展示请求：X条
- 退换货/售后类：X条
- 真实感/色差疑虑：X条
- 其他：X条

【结论】：主要误判模式为 [XXX]，建议将以下内容加入关键词库：[XXX]
```

---

### 模块 C：触发类型效率分析

统计各触发类型的数量、占比和平均回复速度：

| 触发类型 | 数量 | 占比 | 平均回复速度 | 评价 |
|---|---|---|---|---|
| 关键词 | | | | 速度快，稳定 |
| AI问答 | | | | 速度慢，灵活 |
| AI切品 | | | | 用于商品推荐 |
| 意图 | | | | 意图识别触发 |
| 弹商品卡 | | | | 引导点击商品 |
| 兜底 | | | | 兜底话术，需控制占比 |

**重点关注：**
- AI问答 vs 关键词的回复速度差距（AI问答通常慢 3-5 倍，超过 8s 需关注）
- 兜底触发占比若 >10%，说明脚本覆盖不足
- 关键词触发 vs AI问答触发的比例，反映脚本成熟度

---

### 模块 D：回复方式分布分析

| 回复方式 | 说明 | 建议占比 |
|---|---|---|
| 主播+公屏 | 播报+文字同步推送，曝光最大 | 主力方式，>60% |
| 仅主播 | 只有语音播报 | 适合快速简短回复 |
| 仅公屏 | 只推文字，不影响主播语音流 | 适合高频重复问题 |

---

### 模块 E：关键词配置质量分析

从关键词数据 Sheet 分析：

```
关键词有效率 = 有效命中规则次数 / 实际命中次数 × 100%
```

- **有效率 <80%** 的关键词：命中了但没成功触发，检查触发条件是否过严
- **命中次数高但未在列表中** 的词：说明用户常说但脚本没配置，是盲区
- **高频命中词**：说明用户关注点，可加强对应话术深度

---

### 模块 F：MOS 评分趋势分析

适用于有多场次 MOS 数据时，分析数字人质量随时间的变化：

- **互动能力评分**：是衡量数字人直播效果的核心指标，重点追踪
- **人物唇部 + 人物朗读**：技术质量指标，持续低分需提工单
- **MOS 综合分低于 4.0**：触发质量复盘

---

### 模块 G：单用户行为分析（高价值用户识别）

同一场直播中，按用户昵称分组统计：
- 发弹幕次数最多的用户 → 高意向用户，是否获得了有效回复？
- 多次发同一问题未被回复的用户 → 直接指向脚本盲区

---
