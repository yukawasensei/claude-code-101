## 第一部分：开始之前（800字，3分钟）

### 1.1 Claude Code是什么？

用日常语言来说,Claude Code就像是一个24小时在线的技术助手。你不需要懂编程,只需要用自然语言告诉它你想做什么,它就能帮你完成。

**与ChatGPT的关键区别**：ChatGPT只能给建议和代码，Claude Code可以直接在你的电脑上创建文件、运行代码、分析数据

**适用场景**：分析TikTok视频数据找爆款规律、制作可视化仪表盘、整理项目文档生成进度报告、自动化重复性工作

**真实案例**：小王是短视频运营，有50条视频数据。以前花2小时做Excel透视表和图表，现在用Claude Code，10分钟得到完整分析报告。

### 1.2 你需要准备什么

**硬件**：一台Mac电脑
**软件**：推荐安装Warp终端（对新手友好），无需提前安装其他开发工具
**账号**：任意兼容OpenAI API格式的模型服务（Claude API、OpenAI API、国内中转服务等）或本地模型
**心态**：不需要懂编程，愿意尝试新工具，接受迭代式工作方式

## 第二部分：命令行基础（600字，3分钟）

### 2.1 什么是命令行？

命令行就像是电脑的"文字版操作界面"。平时你用鼠标点击文件夹、打开文件,在命令行里,你用文字命令来完成同样的操作。

**为什么需要了解命令行？**
因为Claude Code运行在命令行环境中。不用担心,你只需要掌握3个基础命令就够了。

### 2.1.1 安装 Warp 终端（推荐）

**推荐理由**：界面现代化,支持鼠标操作,有命令提示,对新手友好

**快速安装**：
1. 访问 https://www.warp.dev 下载安装
2. 按 `Command + 空格`,输入 "Warp",回车打开

**备选方案**：使用系统自带的终端（Terminal）
- 按 `Command + 空格`,输入"终端"或"Terminal",回车打开

### 2.2 三个必会命令

#### 1. pwd - 我在哪里？
```bash
pwd
```
显示当前所在的文件夹路径。示例输出：`/Users/你的用户名/Desktop`

#### 2. ls - 这里有什么？
```bash
ls
```
列出当前文件夹里的所有文件和子文件夹。进阶用法：`ls -la`（显示隐藏文件）

#### 3. cd - 去别的地方
```bash
cd Desktop
```
切换到另一个文件夹。特殊路径：
- `cd ~` 回到用户主目录
- `cd ..` 返回上一级目录

### 2.3 跟着做：创建练习文件夹

现在打开终端,跟着输入以下命令：

```bash
# 1. 进入桌面
cd ~/Desktop

# 2. 创建练习文件夹
mkdir claude-practice

# 3. 进入这个文件夹
cd claude-practice

# 4. 确认位置
pwd
```

如果看到类似 `/Users/你的用户名/Desktop/claude-practice` 的输出,说明成功了！

**路径概念**：
- **绝对路径**：从根目录开始的完整路径,如 `/Users/zhangsan/Desktop/my-project`
- **相对路径**：相对于当前位置的路径,如 `./data/report.csv`

**为什么重要？**
Claude Code需要知道文件在哪里才能读取和处理。使用绝对路径最保险。

### 3.1 安装Claude Code

**第一步：安装Node.js**（如果已安装可跳过）

看到版本号说明安装成功！

**常见问题**：
- 提示权限不足 → 使用 `sudo npm install -g @anthropic-ai/claude-code`
- 找不到npm命令 → 重新安装Node.js

### 3.2 使用 CC Switch 快速配置 API

**什么是 CC Switch？**
图形化工具,用来管理 Claude Code 的 API 配置,支持50+预设服务商模板,一键切换。

**安装**：
```bash
# macOS（推荐使用 Homebrew）
brew tap farion1231/ccswitch
brew install --cask cc-switch

# 或访问 https://github.com/farion1231/cc-switch/releases 下载安装包
```

**快速配置**：
1. 打开 CC Switch 应用
2. 点击 "Add Provider",选择预设模板（如 "Claude Official"）
3. 输入你的 API Key
4. 点击 "Enable" 启用配置
5. 配置立即生效（无需重启终端）

**成功提示**：界面显示 `✓ 当前使用: Claude Official (claude-opus-4-6)`

### 7.2 常见问题速查

| 问题 | 解决方案 | 示例命令/提示 |
|------|---------|--------------|
| **Claude说找不到文件** | 检查文件是否存在,确认文件名拼写（注意大小写）,使用绝对路径 | `ls -la` 或 `ls \| grep data` 或提示："请读取 /Users/你的用户名/Desktop/project/data.csv" |
| **分析结果不符合预期** | 提供更多上下文信息,给出参考示例,明确说明哪里不对和期望是什么,分步调试逐个验证 | "请调整分析,我期望看到按月份分组的统计,而不是按天" |
| **生成的图表样式不满意** | 明确说明需要调整的样式细节 | "请调整图表样式：标题字号改为24px,颜色改为#2563eb,图例位置移到右侧,增加网格线" |
| **处理大文件很慢** | 先分析文件大小和结构,考虑分批处理,只读取需要的列,使用采样分析 | "请先分析文件大小,如果超过10MB,只读取前1000行进行采样分析" |
| **需要处理多个相似文件** | 让Claude写脚本批量处理 | "我有10个CSV文件（data1.csv到data10.csv）,每个文件结构相同。请写一个脚本批量处理：读取所有文件,合并数据,统计总计,生成汇总报告" |
