## 第一部分：开始之前（800字，3分钟）

### 1.1 Claude Code是什么？

用日常语言来说,Claude Code就像是一个24小时在线的技术助手。你不需要懂编程,只需要用自然语言告诉它你想做什么,它就能帮你完成。

**与ChatGPT的关键区别**：
- ChatGPT只能给你建议和代码,你还需要自己复制粘贴
- Claude Code可以直接在你的电脑上创建文件、运行代码、分析数据

**适用场景举例**：
- 分析TikTok视频数据,找出爆款规律
- 制作可视化仪表盘,展示运营数据
- 整理项目文档,生成进度报告
- 自动化重复性工作

**真实案例**：
> 小王是一名短视频运营,手上有50条视频的数据表格。她想知道哪个时间段发布视频效果最好,哪类标题更吸引人。以前她要花2小时做Excel透视表和图表,现在用Claude Code,10分钟就能得到一份包含数据分析和可视化图表的完整报告。

### 1.2 你需要准备什么

**硬件要求**：
- 一台Mac电脑（本教程基于macOS）

**软件要求**：
- 推荐安装 Warp 终端（对新手更友好的现代化终端）
- 无需提前安装其他开发工具
- Claude Code会在安装时自动配置所需环境

**账号要求**：
- 任意兼容 OpenAI API 格式的模型服务（如 Claude API、OpenAI API、国内中转服务等）
- 或者使用本地模型服务

**心态准备**：
- 不需要懂编程
- 愿意尝试新工具
- 接受迭代式工作方式（一步步完善）

## 第二部分：命令行基础（1200字，5分钟）

### 2.1 什么是命令行？

命令行就像是电脑的"文字版操作界面"。平时你用鼠标点击文件夹、打开文件,在命令行里,你用文字命令来完成同样的操作。

**为什么需要了解命令行？**
因为Claude Code运行在命令行环境中。不用担心,你只需要掌握5个基础命令就够了。

### 2.1.1 安装 Warp 终端（推荐）

**为什么推荐 Warp？**
- 界面现代化,更像普通软件
- 支持鼠标操作和常规快捷键
- 有命令提示和自动补全
- 对新手更友好

**安装步骤**：
1. 访问 https://www.warp.dev
2. 点击 "Download for Mac"
3. 下载完成后,打开 dmg 文件
4. 将 Warp 拖到应用程序文件夹
5. 首次打开可能需要注册账号（免费）

**如何打开 Warp**：
1. 按 `Command + 空格` 打开Spotlight搜索
2. 输入 "Warp"
3. 回车打开

**如果不想安装 Warp**：
也可以使用系统自带的终端（Terminal）：
1. 按 `Command + 空格` 打开Spotlight搜索
2. 输入"终端"或"Terminal"
3. 回车打开

第一次看到命令行界面不要慌,这就是你的工作环境。

### 2.1.2 终端操作技巧

终端里的操作和普通软件不太一样,这里是一些必须掌握的技巧：

**复制和粘贴**：
- **复制**：选中文字后按 `Command + C`（Warp）或 `Command + C`（Terminal）
- **粘贴**：按 `Command + V`（Warp）或 `Command + V`（Terminal）
- **注意**：在系统终端中,有时需要用 `Command + Shift + C/V`

**全选**：
- Warp：`Command + A`
- Terminal：`Command + A`（选中当前行）

**移动光标**：
- `Control + A`：移动到行首
- `Control + E`：移动到行尾
- `Option + ←/→`：按单词移动
- `←/→`：按字符移动

**删除文字**：
- `Delete`：删除光标前的字符
- `Control + U`：删除光标前的所有内容
- `Control + K`：删除光标后的所有内容
- `Control + W`：删除光标前的一个单词

**历史命令**：
- `↑`：查看上一条命令
- `↓`：查看下一条命令
- `Control + R`：搜索历史命令

**中断命令**：
- `Control + C`：停止当前正在运行的命令

**清屏**：
- `Command + K`（Warp）或 `Control + L`（Terminal）：清空屏幕

**小技巧**：
- 在 Warp 中,你可以像在普通编辑器中一样使用鼠标选择和编辑
- 按 `Tab` 键可以自动补全文件名和命令
- 输入命令时不确定,可以按两次 `Tab` 查看可能的选项

### 2.2 五个必会命令

#### 1. pwd - 我在哪里？
```bash
pwd
```
**作用**：显示当前所在的文件夹路径
**类比**：就像在Finder里查看当前文件夹的位置
**示例输出**：`/Users/你的用户名/Desktop`

#### 2. ls - 这里有什么？
```bash
ls
```
**作用**：列出当前文件夹里的所有文件和子文件夹
**类比**：就像在Finder里查看文件夹内容
**进阶用法**：`ls -la`（显示隐藏文件和详细信息）

#### 3. cd - 去别的地方
```bash
cd Desktop
```
**作用**：切换到另一个文件夹
**类比**：就像在Finder里双击打开文件夹
**特殊路径**：
- `cd ~` 回到用户主目录
- `cd ..` 返回上一级目录

#### 4. mkdir - 创建文件夹
```bash
mkdir my-project
```
**作用**：创建一个新文件夹
**类比**：就像在Finder里右键"新建文件夹"

#### 5. cat - 快速查看文件
```bash
cat README.md
```
**作用**：在终端里显示文件内容
**类比**：就像双击打开文本文件查看

### 2.3 跟着做：创建练习文件夹

现在打开终端,跟着输入以下命令：

```bash
# 1. 进入桌面
cd ~/Desktop

# 2. 创建练习文件夹
mkdir claude-practice

# 3. 进入这个文件夹
cd claude-practice

# 4. 确认位置
pwd
```

如果看到类似 `/Users/你的用户名/Desktop/claude-practice` 的输出,说明成功了！

### 2.4 路径的概念

**绝对路径**：从根目录开始的完整路径
- 示例：`/Users/zhangsan/Desktop/my-project`
- 类比：完整的家庭住址"北京市朝阳区XX街道XX号"

**相对路径**：相对于当前位置的路径
- 示例：`./data/report.csv`（当前文件夹下的data文件夹）
- 类比：相对地址"隔壁楼3单元"

**为什么重要？**
Claude Code需要知道文件在哪里才能读取和处理。使用绝对路径最保险。

## 第三部分：安装和配置（1000字，4分钟）

### 3.1 安装Claude Code

**第一步：安装Node.js**（如果已安装可跳过）
1. 访问 https://nodejs.org
2. 下载LTS版本（长期支持版）
3. 双击安装包,按提示完成安装

**第二步：安装Claude Code**
打开终端（Warp 或 Terminal）,输入：
```bash
npm install -g @anthropic-ai/claude-code
```

等待安装完成（可能需要1-2分钟）。

**第三步：验证安装**
```bash
claude --version
```

如果看到版本号（如 `1.0.0`）,说明安装成功！

**可能遇到的问题**：

**问题1：提示权限不足**
解决方案：在命令前加 `sudo`
```bash
sudo npm install -g @anthropic-ai/claude-code
```
然后输入你的Mac密码（输入时不会显示,这是正常的）。

**问题2：找不到npm命令**
说明Node.js没装好,重新安装Node.js。

### 3.2 使用 CC Switch 快速配置 API

**什么是 CC Switch？**
CC Switch 是一个桌面应用程序,专门用来管理 Claude Code 等 AI 编码工具的 API 配置。它提供了图形化界面,让你可以轻松切换不同的 API 服务商,无需手动编辑配置文件。

**主要优势**：
- 50+ 预设服务商模板（Claude、OpenAI、国内中转等）
- 一键切换 API,无需手动改配置
- Claude Code 支持热切换,无需重启
- 图形化界面,操作简单直观

**安装 CC Switch**：

**macOS 用户（推荐使用 Homebrew）**：
```bash
# 添加 CC Switch 仓库
brew tap farion1231/ccswitch

# 安装 CC Switch
brew install --cask cc-switch
```

**或者直接下载安装**：
1. 访问 https://github.com/farion1231/cc-switch/releases
2. 下载最新版本的安装包：
   - macOS: 下载 `.zip` 文件
   - Windows: 下载 `.msi` 安装包
   - Linux: 下载 `.deb`、`.rpm` 或 `.AppImage`
3. 双击安装

**首次使用**：

**步骤1：启动 CC Switch**
- 在应用程序文件夹中找到 CC Switch
- 双击打开
- 你会看到一个现代化的图形界面

**步骤2：添加 Provider（API 配置）**

点击界面上的 "Add Provider" 按钮,有两种方式：

**方式一：使用预设模板（推荐新手）**

1. 在弹出的窗口中,你会看到 50+ 预设模板
2. 常用选项：
   - **Claude (Anthropic Official)** - Claude 官方 API
   - **OpenAI** - OpenAI 官方 API
   - **AWS Bedrock** - 亚马逊云服务
   - **国内中转服务** - 各种中转服务商
   - **Official Login** - 使用官方账号登录

3. 选择你的服务商（比如选择 "Claude Official"）
4. 在弹出的配置框中输入你的 API Key
5. 点击 "Save" 保存

**方式二：自定义配置（高级用户）**

1. 选择 "Custom" 自定义选项
2. 填写配置信息：
   - **Provider Name**: 给这个配置起个名字（如 "我的本地模型"）
   - **API Base URL**: API 地址（如 `http://localhost:11434/v1`）
   - **API Key**: 你的密钥（本地模型可留空）
   - **Model Name**: 模型名称（如 `llama2`）
3. 点击 "Save" 保存

**步骤3：启用 Provider**

1. 在主界面的 Provider 列表中,找到你刚添加的配置
2. 点击该配置右侧的 "Enable" 按钮
3. 配置立即生效（Claude Code 支持热切换,无需重启终端）

**成功提示**：
界面上会显示当前激活的 Provider,比如：
```
✓ 当前使用: Claude Official (claude-opus-4-6)
```

## 第四部分：基础使用（1000字，4分钟）

### 4.1 启动Claude Code

**重要原则**：在哪个文件夹启动Claude Code,它就在哪个文件夹工作。

**正确的启动流程**：
```bash
# 1. 进入你的项目文件夹
cd ~/Desktop/my-project

# 2. 启动Claude Code
claude
```

**界面说明**：
- 提示符 `You:` 表示等待你输入
- 输入后按回车发送
- Claude的回复会显示在下方
- 如果Claude创建了文件,会显示文件路径

### 4.2 基本对话技巧

**好的提示词特征**：
- 具体明确
- 包含上下文
- 说明预期结果

**对比示例**：

❌ **不好的提示词**：
```
分析数据
```
问题：什么数据？分析什么？输出什么？

✅ **好的提示词**：
```
请读取data.csv文件,分析播放量和点赞量的关系,
生成一个散点图,保存为HTML格式。
```

❌ **不好的提示词**：
```
做个网页
```

✅ **好的提示词**：
```
创建一个TikTok数据仪表盘页面,包含：
1. 顶部显示总播放量、总点赞量、平均转化率
2. 中间显示播放量趋势折线图
3. 底部显示视频类型分布饼图
使用蓝白配色,现代简洁风格。
```

### 4.3 常用命令

在Claude Code对话中,以 `/` 开头的是特殊命令：

**基础命令**：
- `/help` - 查看帮助信息和所有可用命令
- `/clear` - 清空当前对话历史（释放上下文空间）
- `/exit` - 退出Claude Code
- 上下箭头键 - 查看和重用历史输入

**会话管理**：
- `/init` - 优化和初始化项目的 CLAUDE.md 配置文件
- `/resume` - 恢复之前的会话（继续上次未完成的工作）

**模型和性能**：
- `/model` - 查看或切换使用的AI模型
- `/fast` - 切换快速模式（使用相同模型但输出更快）

**使用技巧**：
- 使用 `/init` 当你想为项目创建或优化 CLAUDE.md 配置时
- 使用 `/resume` 当你想继续之前的分析或项目时
- 使用 `/clear` 当对话太长影响理解时（但会丢失上下文）
- 使用 `/fast` 当你需要快速迭代时

### 4.4 文件操作理解

**Claude Code如何工作**：
1. 你提出需求
2. Claude分析需求,决定需要创建/修改哪些文件
3. Claude执行操作并告诉你结果
4. 你可以在Finder或用命令查看文件

**查看Claude创建的文件**：
```bash
# 方法1：用命令行
ls -la

# 方法2：用Finder
open .
```

**重要提示**：
- Claude只能操作当前工作目录及其子目录的文件
- 如果文件在别的地方,需要提供完整路径
- Claude不会删除文件,除非你明确要求

## 第五部分：Case 1 - TikTok视频数据分析与可视化仪表盘（1800字，7分钟）

### 5.1 案例背景

**场景设定**：
你是一名短视频运营,负责公司的TikTok账号。过去两个月发布了50条视频,现在需要分析数据,找出爆款规律,为下个月的内容策略提供依据。

**手上的数据**：
一份Excel表格,包含50条视频的运营数据：
- 视频ID、标题、发布时间
- 播放量、点赞量、评论量、分享量
- 视频时长、视频类型
- 成单量（转化数据）

**老板的需求**：
1. 哪个时间段发布效果最好？
2. 哪类视频转化率最高？
3. 播放量和转化之间的关系？
4. 做一个可视化仪表盘,下周例会要用

**学习目标**：
- 掌握数据探索和清洗流程
- 学会多维度统计分析
- 制作交互式可视化图表
- 生成专业的分析报告

### 5.2 准备工作

**Step 1：创建项目文件夹**
```bash
cd ~/Desktop
mkdir tiktok-analysis
cd tiktok-analysis
```

**Step 2：准备数据文件**

在真实场景中,你会从 TikTok 后台或第三方工具导出数据。为了方便学习,我们假设你已经有了一份 `tiktok_data.csv` 文件。

**数据文件说明**：
这份 CSV 文件包含 50 条视频的运营数据,字段包括：
- `video_id`: 视频ID
- `title`: 视频标题
- `publish_date`: 发布日期
- `publish_hour`: 发布小时（0-23）
- `views`: 播放量
- `likes`: 点赞量
- `comments`: 评论量
- `shares`: 分享量
- `duration`: 视频时长（秒）
- `video_type`: 视频类型（教程、剧情、测评、好物推荐）
- `conversions`: 成单量

**如何获取数据文件**：
1. 从 TikTok 创作者中心导出数据
2. 或从第三方数据分析工具导出
3. 或使用本教程提供的示例数据（见附录C）

**将数据文件放到项目文件夹**：
```bash
# 方法1：用 Finder 拖拽文件到 tiktok-analysis 文件夹

# 方法2：如果文件在下载文件夹,用命令复制
cp ~/Downloads/tiktok_data.csv ~/Desktop/tiktok-analysis/

# 验证文件是否存在
ls -la
```

你应该看到 `tiktok_data.csv` 文件。

**启动 Claude Code**：
```bash
claude
```

### 5.3 数据探索

现在开始分析数据。第一步是了解数据的基本情况。

**对话示例**：
```
你：请读取tiktok_data.csv,给我一个数据概览,包括：
1. 总共多少条记录
2. 每个字段的数据类型
3. 是否有缺失值
4. 播放量、点赞量、转化量的基本统计（最小值、最大值、平均值、中位数）
```

**Claude会输出类似**：
```
数据概览：
- 总记录数：50条
- 时间跨度：2026-01-01 至 2026-02-28

字段统计：
播放量：
  - 平均值：15,234
  - 中位数：12,500
  - 最大值：48,900
  - 最小值：1,200

转化量：
  - 平均值：8.5
  - 中位数：6
  - 最大值：45
  - 最小值：0

数据质量：无缺失值,数据完整
```

**学习要点**：
- 数据探索是分析的第一步
- 了解数据的基本情况,才能做正确的分析
- Claude 可以自动处理 CSV 文件,无需你懂 pandas
- 如果发现数据质量问题（如缺失值、异常值）,可以让 Claude 帮你清洗

### 5.4 多维度分析

#### 分析1：最佳发布时间

**对话示例**：
```
你：请分析发布时间对播放量的影响。
按小时统计平均播放量,找出播放量最高的3个时间段。
生成一个柱状图,保存为HTML格式。
```

**Claude会做什么**：
1. 按小时分组统计平均播放量
2. 创建交互式柱状图（使用Chart.js）
3. 生成 `publish_time_analysis.html`
4. 给出结论

**预期结论**：
```
最佳发布时间：
1. 20:00-21:00（晚8点）- 平均播放量 28,500
2. 19:00-20:00（晚7点）- 平均播放量 25,300
3. 21:00-22:00（晚9点）- 平均播放量 22,100

建议：优先在晚上7-9点发布视频
```

**查看图表**：
```bash
open publish_time_analysis.html
```

#### 分析2：视频类型转化率

**对话示例**：
```
你：请分析不同视频类型的转化效果。
计算每种类型的：
1. 平均播放量
2. 平均转化量
3. 转化率（转化量/播放量*10000）

生成对比表格和柱状图。
```

**Claude会输出**：
```
视频类型转化分析：

类型      平均播放量  平均转化量  转化率(‱)
教程      18,500      15.2       8.2
测评      16,200      10.5       6.5
好物推荐  14,800      8.3        5.6
剧情      12,300      4.1        3.3

结论：教程类视频转化率最高,是剧情类的2.5倍
```

#### 分析3：播放量与转化关系

**对话示例**：
```
你：请分析播放量和转化量的关系。
生成散点图,并计算相关系数。
如果有明显的规律,请说明。
```

**Claude会做什么**：
- 创建散点图
- 计算相关系数
- 识别异常值（播放量高但转化低的视频）
- 给出优化建议

**预期输出**：
```
相关性分析：
- 相关系数：0.78（强正相关）
- 结论：播放量越高,转化量越高

但发现3个异常值：
- 视频#23：播放量45000,但转化量只有2
```

### 7.3 Claude Code 高级技巧

这些技巧可以显著提升你使用 Claude Code 的效率和体验。

**技巧1：YOLO模式 - 跳过权限确认**
```bash

```
**说明**：
- 启动时添加此参数，Claude Code 会自动执行所有操作，不再弹出权限确认
- 适合你完全信任 Claude 的场景，可以大幅提升工作效率
- ⚠️ 注意：使用此模式时要格外小心，Claude 可能会执行删除、修改等操作

**使用场景**：
- 处理大量重复性任务
- 你已经明确知道要做什么
- 在测试环境中快速迭代

**技巧2：Plan 计划模式 - 先规划再执行**
```bash
# 启动 Claude Code 后，按 Shift+Tab 切换到 Plan 模式
```
**说明**：
- Plan 模式让 Claude 先制定详细的执行计划，等你确认后再执行
- 适合复杂任务，可以避免走弯路
- 按 `Shift+Tab` 可以在普通模式和 Plan 模式之间切换

**使用场景**：
- 大型重构或架构调整
- 不确定最佳方案的任务
- 需要评估多个方案的场景

**技巧3：定期清理上下文**
```bash
# 在 Claude Code 对话中
/clear
```
**说明**：
- 每次完成一个独立任务后，使用 `/clear` 清理上下文
- 避免上下文过长影响 Claude 的理解和响应速度
- 清理前记得保存重要信息到文件或 CLAUDE.md

**最佳实践**：
- 完成一个分析任务后 → `/clear` → 开始下一个任务
- 生成报告后 → `/clear` → 开始新的分析
- 感觉 Claude 理解出现偏差时 → `/clear` → 重新开始

### Case 1：TikTok数据设计

**数据规模**：50条记录

**字段设计**：
```csv
video_id,title,publish_date,publish_hour,views,likes,comments,shares,duration,video_type,conversions
V001,3分钟学会做红烧肉,2026-01-05,19,28500,3420,285,570,180,教程,35
V002,今天又是打工人的一天,2026-01-06,12,12300,1476,123,246,45,剧情,8
...
```

**数据规律**：
1. **发布时间影响**：
   - 19:00-21:00发布的视频,播放量在20000-50000
   - 其他时段,播放量在5000-20000
   
2. **视频类型影响**：
   - 教程类：转化率6-10‱
   - 测评类：转化率4-8‱
   - 好物推荐：转化率3-7‱
   - 剧情类：转化率1-5‱

3. **互动数据关系**：
   - 点赞量 = 播放量 × (5%-15%)
   - 评论量 = 播放量 × (1%-3%)
   - 分享量 = 播放量 × (0.5%-2%)

4. **转化规律**：
   - 基础转化 = 播放量 × 转化率
   - 加入随机波动（±30%）
   - 3-5个异常值（高播放低转化）

**标题示例**：
- 教程类："3分钟学会XX"、"新手必看XX教程"、"手把手教你XX"
- 测评类："XX真的值得买吗"、"XX深度测评"、"XX使用一个月后"
- 好物推荐："这个XX太好用了"、"XX好物分享"、"XX必买清单"
- 剧情类："今天又是XX的一天"、"XX的日常"、"XX搞笑瞬间"
