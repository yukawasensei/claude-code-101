## 第一部分：开始之前（800字，3分钟）

### 开场白

想象一下,老板给你50条视频数据,要你分析出哪个时间段发布效果最好、哪类内容转化率最高,还要做成可视化报告,下周例会要用。

以前你可能要花2-4小时:打开Excel、做透视表、画图表、写分析、调格式...累得要死,结果还不一定让老板满意。

现在有了 Claude Code,10分钟搞定。而且不需要懂编程。

听我一句劝,这比你想象的简单。你只需要学会"说人话"——用自然语言告诉 Claude Code 你想要什么,它就能自动帮你完成所有分析工作。

### 1.1 Claude Code是什么?

Claude Code 就像一个24小时在线的技术助手。你不需要懂编程,只需要用自然语言告诉它你想做什么,它就能帮你完成。

**与ChatGPT的关键区别**：
- ChatGPT只能给你建议和代码,你还需要自己复制粘贴
- Claude Code可以直接在你的电脑上创建文件、运行代码、分析数据

**适用场景举例**：
- 分析TikTok视频数据,找出爆款规律
- 制作可视化仪表盘,展示运营数据
- 整理项目文档,生成进度报告
- 自动化重复性工作

**Wow moment**:
一个提示词,2分钟,自动生成:
- 10个维度的深度分析报告
- 专业的可视化网页
- 具体可执行的优化建议

不用写代码,不用学Python,不用懂数据分析。

### 1.2 你需要准备什么

**硬件要求**：
- 一台Mac电脑（本教程基于macOS）

**软件要求**：
- 推荐安装 Warp 终端（对新手更友好的现代化终端）
- 无需提前安装其他开发工具
- Claude Code会在安装时自动配置所需环境

**账号要求**：
- 任意兼容 OpenAI API 格式的模型服务（如 Claude API、OpenAI API、国内中转服务等）

**心态准备**：
- 不需要懂编程
- 愿意尝试新工具
- 接受迭代式工作方式（一步步完善）

## 第二部分：快速上手（600字，3分钟）

### 2.1 什么是命令行?

命令行就像是电脑的"文字版操作界面"。平时你用鼠标点击文件夹、打开文件,在命令行里,你用文字命令来完成同样的操作。

不用担心,你只需要掌握5个基础命令就够了。

**如何打开终端**:
1. 按 `Command + 空格` 打开Spotlight搜索
2. 输入"终端"或"Terminal"
3. 回车打开

第一次看到命令行界面不要慌,这就是你的工作环境。

**友情提示**: 推荐安装 Warp 终端(https://www.warp.dev),界面更现代化,对新手更友好。但系统自带的终端也完全够用。

### 2.2 五个必会命令

#### 1. pwd - 我在哪里？
```bash
pwd
```
**作用**：显示当前所在的文件夹路径
**类比**：就像在Finder里查看当前文件夹的位置
**示例输出**：`/Users/你的用户名/Desktop`

#### 2. ls - 这里有什么？
```bash
ls
```
**作用**：列出当前文件夹里的所有文件和子文件夹
**类比**：就像在Finder里查看文件夹内容
**进阶用法**：`ls -la`（显示隐藏文件和详细信息）

#### 3. cd - 去别的地方
```bash
cd Desktop
```
**作用**：切换到另一个文件夹
**类比**：就像在Finder里双击打开文件夹
**特殊路径**：
- `cd ~` 回到用户主目录
- `cd ..` 返回上一级目录

#### 4. mkdir - 创建文件夹
```bash
mkdir my-project
```
**作用**：创建一个新文件夹
**类比**：就像在Finder里右键"新建文件夹"

#### 5. cat - 快速查看文件
```bash
cat README.md
```
**作用**：在终端里显示文件内容
**类比**：就像双击打开文本文件查看

### 2.3 跟着做：创建练习文件夹

现在打开终端,跟着输入以下命令：

```bash
# 1. 进入桌面
cd ~/Desktop

# 2. 创建练习文件夹
mkdir claude-practice

# 3. 进入这个文件夹
cd claude-practice

# 4. 确认位置
pwd
```

如果看到类似 `/Users/你的用户名/Desktop/claude-practice` 的输出,说明成功了！

**记住**: Claude Code 就像一个新来的同事,你得告诉它文件在哪儿。使用完整路径最保险。

## 第三部分：安装和配置（1000字，4分钟）

### 3.1 安装Claude Code

**第一步：安装Node.js**（如果已安装可跳过）
1. 访问 https://nodejs.org
2. 下载LTS版本（长期支持版）
3. 双击安装包,按提示完成安装

**第二步：安装Claude Code**
打开终端（Warp 或 Terminal）,输入：
```bash
npm install -g @anthropic-ai/claude-code
```

等待安装完成（可能需要1-2分钟）。

**第三步：验证安装**
```bash
claude --version
```

如果看到版本号（如 `1.0.0`）,说明安装成功！

**可能遇到的问题**：

**问题1：提示权限不足**
解决方案：在命令前加 `sudo`
```bash
sudo npm install -g @anthropic-ai/claude-code
```
然后输入你的Mac密码（输入时不会显示,这是正常的）。

**问题2：找不到npm命令**
说明Node.js没装好,重新安装Node.js。

### 3.2 使用 CC Switch 快速配置 API

**什么是 CC Switch？**
CC Switch 是一个桌面应用程序,专门用来管理 Claude Code 等 AI 编码工具的 API 配置。它提供了图形化界面,让你可以轻松切换不同的 API 服务商,无需手动编辑配置文件。

**主要优势**：
- 50+ 预设服务商模板（Claude、OpenAI、国内中转等）
- 一键切换 API,无需手动改配置
- Claude Code 支持热切换,无需重启
- 图形化界面,操作简单直观

**安装 CC Switch**：

**macOS 用户（推荐使用 Homebrew）**：
```bash
# 添加 CC Switch 仓库
brew tap farion1231/ccswitch

# 安装 CC Switch
brew install --cask cc-switch
```

**或者直接下载安装**：
1. 访问 https://github.com/farion1231/cc-switch/releases
2. 下载最新版本的安装包：
   - macOS: 下载 `.zip` 文件
   - Windows: 下载 `.msi` 安装包
   - Linux: 下载 `.deb`、`.rpm` 或 `.AppImage`
3. 双击安装

**首次使用**：

**步骤1：启动 CC Switch**
- 在应用程序文件夹中找到 CC Switch
- 双击打开
- 你会看到一个现代化的图形界面

**步骤2：添加 Provider（API 配置）**

点击界面上的 "Add Provider" 按钮,有两种方式：

**方式一：使用预设模板（推荐新手）**

1. 在弹出的窗口中,你会看到 50+ 预设模板
2. 常用选项：
   - **Claude (Anthropic Official)** - Claude 官方 API
   - **OpenAI** - OpenAI 官方 API
   - **AWS Bedrock** - 亚马逊云服务
   - **国内中转服务** - 各种中转服务商
   - **Official Login** - 使用官方账号登录

3. 选择你的服务商（比如选择 "Claude Official"）
4. 在弹出的配置框中输入你的 API Key
5. 点击 "Save" 保存

**方式二：自定义配置（高级用户）**

1. 选择 "Custom" 自定义选项
2. 填写配置信息：
   - **Provider Name**: 给这个配置起个名字（如 "我的本地模型"）
   - **API Base URL**: API 地址（如 `http://localhost:11434/v1`）
   - **API Key**: 你的密钥（本地模型可留空）
   - **Model Name**: 模型名称（如 `llama2`）
3. 点击 "Save" 保存

**步骤3：启用 Provider**

1. 在主界面的 Provider 列表中,找到你刚添加的配置
2. 点击该配置右侧的 "Enable" 按钮
3. 配置立即生效（Claude Code 支持热切换,无需重启终端）

**成功提示**：
界面上会显示当前激活的 Provider,比如：
```
✓ 当前使用: Claude Official (claude-opus-4-6)
```

### 4.1 启动Claude Code

**重要原则**：在哪个文件夹启动Claude Code,它就在哪个文件夹工作。

**正确的启动流程**：
```bash

# 2. 启动Claude Code

```

**界面说明**：
- 提示符 `You:` 表示等待你输入
- 输入后按回车发送
- Claude的回复会显示在下方
- 如果Claude创建了文件,会显示文件路径

### 4.2 第一个简单任务

让我们试试第一个简单任务:

```
你: 读取当前目录的文件列表,告诉我有哪些文件
```

恭喜你,你已经完成了第一次对话!

### 4.3 三个常见错误(精简版)

**错误1: 一次性说太多**
- 问题: 一个提示词包含10个不相关的任务
- 后果: Claude 理解混乱,结果不符合预期
- 正确做法: 一次专注一个任务,完成后再进行下一个

**错误2: 不管理上下文**
- 问题: 对话太长,从不使用 `/clear`
- 后果: Claude 理解出现偏差,响应变慢
- 正确做法: 完成一个任务后使用 `/clear` 清理上下文

**错误3: 不利用 CLAUDE.md**
- 问题: 每次都重复说明项目规范
- 后果: 浪费时间,容易遗漏
- 正确做法: 将项目规范写入 CLAUDE.md,Claude 会自动读取

### 4.4 基本对话技巧

**好的提示词特征**：
- 具体明确
- 包含上下文
- 说明预期结果

**对比示例**：

❌ **不好的提示词**：
```
分析数据
```
问题：什么数据？分析什么？输出什么？

✅ **好的提示词**：
```
请读取data.csv文件,分析播放量和点赞量的关系,
生成一个散点图,保存为HTML格式。
```

### 4.5 常用命令

在Claude Code对话中,以 `/` 开头的是特殊命令：

**基础命令**：
- `/help` - 查看帮助信息
- `/clear` - 清空对话历史
- `/exit` - 退出Claude Code

**会话管理**：
- `/init` - 初始化项目的 CLAUDE.md 配置文件
- `/resume` - 恢复之前的会话

**模型和性能**：
- `/model` - 查看或切换AI模型
- `/fast` - 切换快速模式

### 5.3 一个提示词完成所有分析

这是本教程的核心部分。你将学会如何编写一个完整的提示词,让 Claude Code 自动完成所有分析工作,包括生成深度分析报告和可视化网页。

**提示词编写原则**：
1. **清晰**：明确说明你想要什么
2. **具体**：详细描述每个部分的要求
3. **完整**：一次性说明所有需求,避免多次对话

**完整提示词示例**：

```
我有一份直播和短视频运营数据（tiktok_data.csv），包含50场直播和短视频的完整数据。

请使用 /live-ops-analytics skill 生成一份深度分析报告，包含：

1. 数据概览和关键指标
   - 总播放量、总GMV、平均转化率、平均ROI
   - 数据时间范围和样本量

2. 直播间表现分析
   - 场观(UV)分布和趋势
   - 在线峰值和人均停留时长
   - 下单转化率分析
   - 识别高转化直播的特征

3. 短视频效果分析
   - 播放量、互动率(点赞率、评论率、分享率)
   - 不同视频类型的表现对比
   - 发布时间对效果的影响
   - 视频时长与完播率的关系

4. ROI和投流分析
   - 广告花费分布
   - CPM(千次观看成本)分析
   - ROI排名和优化建议
   - 不同内容类型的投放效率

5. 用户互动分析
   - 弹幕数、礼物数分布
   - 关注转化率分析
   - 互动质量与转化的关系
   - 高价值用户特征

6. 优化建议和下月策略
   - 内容策略(类型、时长、发布时间)
   - 投流策略(预算分配、出价优化)
   - 转化优化(引导话术、优惠设计)
   - 具体执行计划和预期效果

报告要求：
- 适合给老板看的管理层汇报风格
- 每个建议都要有数据支撑
- 建议要具体可执行
- 包含预期效果预测

然后使用 /frontend-design skill 生成一个现代化的可视化网页，包含：

1. 顶部核心指标卡片（6-8个关键指标）
   - 总播放量、总GMV、平均转化率
   - 平均ROI、平均CPM、关注转化率
   - 使用卡片式设计,数字突出显示

2. 多维度数据图表
   - 发布时间vs播放量柱状图
   - 视频类型转化率对比图
   - ROI分布散点图
   - 转化漏斗图
   - 内容表现趋势折线图
   - 投流效率对比图

3. 数据表格
   - 高ROI内容Top 10
   - 待优化内容列表
   - 支持排序和筛选

4. 响应式布局
   - 适配桌面和移动端
   - 图表支持交互(悬停显示详细数据)

设计要求：
- 现代简洁风格
- 蓝白配色(主色#2563eb,辅色#60a5fa)
- 高质量的视觉呈现
- 专业的数据可视化

文件名：
- 报告：analysis_report.md
- 网页：dashboard.html
```

**复制上面的提示词,粘贴到 Claude Code 中,按回车发送。**

等待2-3分钟,Claude Code 会自动完成所有分析工作。

## 第六部分:三个关键技巧(500字,2分钟)

### 6.1 提示词要具体

不要说"分析数据",要说"读取data.csv,分析播放量和点赞量的关系,生成散点图"。

**为什么重要**:
- 具体的提示词让 Claude 知道你要什么
- 减少来回沟通的次数
- 提高输出质量

**示例**:
```
❌ 分析这个文件
✅ 读取sales_data.csv,统计每个月的销售额,找出Top 3月份,生成柱状图
```

### 6.2 管理好上下文

对话太长会影响 Claude 的理解。完成一个任务后,使用 `/clear` 清理上下文。

**什么时候清理**:
- 完成一个分析任务后
- 生成报告后准备开始新任务
- 感觉 Claude 理解出现偏差时

**记住**: 清理前保存重要信息到文件或 CLAUDE.md

### 6.3 利用 CLAUDE.md

CLAUDE.md 是 Claude Code 的配置文件,会自动读取。将项目规范、编码风格、注意事项写入这个文件,Claude 会在每次对话时参考。

**示例内容**:
```markdown
# 项目规范

## 数据分析规范
- 所有图表使用 Chart.js
- 配色方案:蓝色系 (#2563eb)
- 报告格式:Markdown

## 注意事项
- 数据文件统一放在 data/ 目录
- 生成的报告保存到 reports/ 目录
```

**好处**:
- 不用每次都重复说明规范
- 确保输出风格一致
- 团队成员共享配置
