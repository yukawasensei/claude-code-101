# Claude Code 101 从入门到入坟

## 教程定位
- **目标用户**：运营、产品经理、数据分析等非技术背景人员
- **阅读时长**：35分钟（约9000字）
- **学习目标**：从零开始掌握Claude Code,学会编写提示词让 Claude Code 自动生成高质量数据分析报告和可视化网页

## 第一部分：开始之前（800字，3分钟）

### 1.1 Claude Code是什么？

用日常语言来说,Claude Code就像是一个24小时在线的技术助手。你不需要懂编程,只需要用自然语言告诉它你想做什么,它就能帮你完成。

**与ChatGPT的关键区别**：
- ChatGPT只能给你建议和代码,你还需要自己复制粘贴
- Claude Code可以直接在你的电脑上创建文件、运行代码、分析数据

**适用场景举例**：
- 分析TikTok视频数据,找出爆款规律
- 制作可视化仪表盘,展示运营数据
- 整理项目文档,生成进度报告
- 自动化重复性工作

**真实案例**：
> 小王是一名短视频运营,手上有50条视频的数据表格。她想知道哪个时间段发布视频效果最好,哪类标题更吸引人。以前她要花2小时做Excel透视表和图表,现在用Claude Code,10分钟就能得到一份包含数据分析和可视化图表的完整报告。

### 1.2 你需要准备什么

**硬件要求**：
- 一台Mac电脑（本教程基于macOS）

**软件要求**：
- 推荐安装 Warp 终端（对新手更友好的现代化终端）
- 无需提前安装其他开发工具
- Claude Code会在安装时自动配置所需环境

**账号要求**：
- 任意兼容 OpenAI API 格式的模型服务（如 Claude API、OpenAI API、国内中转服务等）
- 或者使用本地模型服务

**心态准备**：
- 不需要懂编程
- 愿意尝试新工具
- 接受迭代式工作方式（一步步完善）

## 第二部分：命令行基础（1200字，5分钟）

### 2.1 什么是命令行？

命令行就像是电脑的"文字版操作界面"。平时你用鼠标点击文件夹、打开文件,在命令行里,你用文字命令来完成同样的操作。

**为什么需要了解命令行？**
因为Claude Code运行在命令行环境中。不用担心,你只需要掌握5个基础命令就够了。

### 2.1.1 安装 Warp 终端（推荐）

**为什么推荐 Warp？**
- 界面现代化,更像普通软件
- 支持鼠标操作和常规快捷键
- 有命令提示和自动补全
- 对新手更友好

**安装步骤**：
1. 访问 https://www.warp.dev
2. 点击 "Download for Mac"
3. 下载完成后,打开 dmg 文件
4. 将 Warp 拖到应用程序文件夹
5. 首次打开可能需要注册账号（免费）

**如何打开 Warp**：
1. 按 `Command + 空格` 打开Spotlight搜索
2. 输入 "Warp"
3. 回车打开

**如果不想安装 Warp**：
也可以使用系统自带的终端（Terminal）：
1. 按 `Command + 空格` 打开Spotlight搜索
2. 输入"终端"或"Terminal"
3. 回车打开

第一次看到命令行界面不要慌,这就是你的工作环境。

### 2.1.2 终端操作技巧

终端里的操作和普通软件不太一样,这里是一些必须掌握的技巧：

**复制和粘贴**：
- **复制**：选中文字后按 `Command + C`（Warp）或 `Command + C`（Terminal）
- **粘贴**：按 `Command + V`（Warp）或 `Command + V`（Terminal）
- **注意**：在系统终端中,有时需要用 `Command + Shift + C/V`

**全选**：
- Warp：`Command + A`
- Terminal：`Command + A`（选中当前行）

**移动光标**：
- `Control + A`：移动到行首
- `Control + E`：移动到行尾
- `Option + ←/→`：按单词移动
- `←/→`：按字符移动

**删除文字**：
- `Delete`：删除光标前的字符
- `Control + U`：删除光标前的所有内容
- `Control + K`：删除光标后的所有内容
- `Control + W`：删除光标前的一个单词

**历史命令**：
- `↑`：查看上一条命令
- `↓`：查看下一条命令
- `Control + R`：搜索历史命令

**中断命令**：
- `Control + C`：停止当前正在运行的命令

**清屏**：
- `Command + K`（Warp）或 `Control + L`（Terminal）：清空屏幕

**小技巧**：
- 在 Warp 中,你可以像在普通编辑器中一样使用鼠标选择和编辑
- 按 `Tab` 键可以自动补全文件名和命令
- 输入命令时不确定,可以按两次 `Tab` 查看可能的选项

### 2.2 五个必会命令

#### 1. pwd - 我在哪里？
```bash
pwd
```
**作用**：显示当前所在的文件夹路径
**类比**：就像在Finder里查看当前文件夹的位置
**示例输出**：`/Users/你的用户名/Desktop`

#### 2. ls - 这里有什么？
```bash
ls
```
**作用**：列出当前文件夹里的所有文件和子文件夹
**类比**：就像在Finder里查看文件夹内容
**进阶用法**：`ls -la`（显示隐藏文件和详细信息）

#### 3. cd - 去别的地方
```bash
cd Desktop
```
**作用**：切换到另一个文件夹
**类比**：就像在Finder里双击打开文件夹
**特殊路径**：
- `cd ~` 回到用户主目录
- `cd ..` 返回上一级目录

#### 4. mkdir - 创建文件夹
```bash
mkdir my-project
```
**作用**：创建一个新文件夹
**类比**：就像在Finder里右键"新建文件夹"

#### 5. cat - 快速查看文件
```bash
cat README.md
```
**作用**：在终端里显示文件内容
**类比**：就像双击打开文本文件查看

### 2.3 跟着做：创建练习文件夹

现在打开终端,跟着输入以下命令：

```bash
# 1. 进入桌面
cd ~/Desktop

# 2. 创建练习文件夹
mkdir claude-practice

# 3. 进入这个文件夹
cd claude-practice

# 4. 确认位置
pwd
```

如果看到类似 `/Users/你的用户名/Desktop/claude-practice` 的输出,说明成功了！

### 2.4 路径的概念

**绝对路径**：从根目录开始的完整路径
- 示例：`/Users/zhangsan/Desktop/my-project`
- 类比：完整的家庭住址"北京市朝阳区XX街道XX号"

**相对路径**：相对于当前位置的路径
- 示例：`./data/report.csv`（当前文件夹下的data文件夹）
- 类比：相对地址"隔壁楼3单元"

**为什么重要？**
Claude Code需要知道文件在哪里才能读取和处理。使用绝对路径最保险。

## 第三部分：安装和配置（1000字，4分钟）

### 3.1 安装Claude Code

**第一步：安装Node.js**（如果已安装可跳过）
1. 访问 https://nodejs.org
2. 下载LTS版本（长期支持版）
3. 双击安装包,按提示完成安装

**第二步：安装Claude Code**
打开终端（Warp 或 Terminal）,输入：
```bash
npm install -g @anthropic-ai/claude-code
```

等待安装完成（可能需要1-2分钟）。

**第三步：验证安装**
```bash
claude --version
```

如果看到版本号（如 `1.0.0`）,说明安装成功！

**可能遇到的问题**：

**问题1：提示权限不足**
解决方案：在命令前加 `sudo`
```bash
sudo npm install -g @anthropic-ai/claude-code
```
然后输入你的Mac密码（输入时不会显示,这是正常的）。

**问题2：找不到npm命令**
说明Node.js没装好,重新安装Node.js。

### 3.2 使用 CC Switch 快速配置 API

**什么是 CC Switch？**
CC Switch 是一个桌面应用程序,专门用来管理 Claude Code 等 AI 编码工具的 API 配置。它提供了图形化界面,让你可以轻松切换不同的 API 服务商,无需手动编辑配置文件。

**主要优势**：
- 50+ 预设服务商模板（Claude、OpenAI、国内中转等）
- 一键切换 API,无需手动改配置
- Claude Code 支持热切换,无需重启
- 图形化界面,操作简单直观

**安装 CC Switch**：

**macOS 用户（推荐使用 Homebrew）**：
```bash
# 添加 CC Switch 仓库
brew tap farion1231/ccswitch

# 安装 CC Switch
brew install --cask cc-switch
```

**或者直接下载安装**：
1. 访问 https://github.com/farion1231/cc-switch/releases
2. 下载最新版本的安装包：
   - macOS: 下载 `.zip` 文件
   - Windows: 下载 `.msi` 安装包
   - Linux: 下载 `.deb`、`.rpm` 或 `.AppImage`
3. 双击安装

**首次使用**：

**步骤1：启动 CC Switch**
- 在应用程序文件夹中找到 CC Switch
- 双击打开
- 你会看到一个现代化的图形界面

**步骤2：添加 Provider（API 配置）**

点击界面上的 "Add Provider" 按钮,有两种方式：

**方式一：使用预设模板（推荐新手）**

1. 在弹出的窗口中,你会看到 50+ 预设模板
2. 常用选项：
   - **Claude (Anthropic Official)** - Claude 官方 API
   - **OpenAI** - OpenAI 官方 API
   - **AWS Bedrock** - 亚马逊云服务
   - **国内中转服务** - 各种中转服务商
   - **Official Login** - 使用官方账号登录

3. 选择你的服务商（比如选择 "Claude Official"）
4. 在弹出的配置框中输入你的 API Key
5. 点击 "Save" 保存

**方式二：自定义配置（高级用户）**

1. 选择 "Custom" 自定义选项
2. 填写配置信息：
   - **Provider Name**: 给这个配置起个名字（如 "我的本地模型"）
   - **API Base URL**: API 地址（如 `http://localhost:11434/v1`）
   - **API Key**: 你的密钥（本地模型可留空）
   - **Model Name**: 模型名称（如 `llama2`）
3. 点击 "Save" 保存

**步骤3：启用 Provider**

1. 在主界面的 Provider 列表中,找到你刚添加的配置
2. 点击该配置右侧的 "Enable" 按钮
3. 配置立即生效（Claude Code 支持热切换,无需重启终端）

**成功提示**：
界面上会显示当前激活的 Provider,比如：
```
✓ 当前使用: Claude Official (claude-opus-4-6)
```

## 第四部分：基础使用（1000字，4分钟）

### 4.1 启动Claude Code

**重要原则**：在哪个文件夹启动Claude Code,它就在哪个文件夹工作。

**正确的启动流程**：
```bash
# 1. 进入你的项目文件夹
cd ~/Desktop/my-project

```

**界面说明**：
- 提示符 `You:` 表示等待你输入
- 输入后按回车发送
- Claude的回复会显示在下方

### 4.2 基本对话技巧

**好的提示词特征**：
- 具体明确
- 包含上下文
- 说明预期结果

**对比示例**：

❌ **不好的提示词**：
```
分析数据
```
问题：什么数据？分析什么？输出什么？

✅ **好的提示词**：
```
请读取data.csv文件,分析播放量和点赞量的关系,
生成一个散点图,保存为HTML格式。
```

❌ **不好的提示词**：
```
做个网页
```

✅ **好的提示词**：
```
创建一个TikTok数据仪表盘页面,包含：
1. 顶部显示总播放量、总点赞量、平均转化率
2. 中间显示播放量趋势折线图
3. 底部显示视频类型分布饼图
使用蓝白配色,现代简洁风格。
```

### 4.3 常用命令

在Claude Code对话中,以 `/` 开头的是特殊命令：

**基础命令**：
- `/help` - 查看帮助信息和所有可用命令
- `/clear` - 清空当前对话历史（释放上下文空间）
- `/exit` - 退出Claude Code
- 上下箭头键 - 查看和重用历史输入

**会话管理**：
- `/init` - 优化和初始化项目的 CLAUDE.md 配置文件
- `/resume` - 恢复之前的会话（继续上次未完成的工作）

**模型和性能**：
- `/model` - 查看或切换使用的AI模型
- `/fast` - 切换快速模式（使用相同模型但输出更快）

**使用技巧**：
- 使用 `/init` 当你想为项目创建或优化 CLAUDE.md 配置时
- 使用 `/resume` 当你想继续之前的分析或项目时
- 使用 `/clear` 当对话太长影响理解时（但会丢失上下文）
- 使用 `/fast` 当你需要快速迭代时

### 4.4 文件操作理解

**Claude Code如何工作**：
1. 你提出需求
2. Claude分析需求,决定需要创建/修改哪些文件
3. Claude执行操作并告诉你结果
4. 你可以在Finder或用命令查看文件

**查看Claude创建的文件**：
```bash
# 方法1：用命令行
ls -la

# 方法2：用Finder
open .
```

**重要提示**：
- Claude只能操作当前工作目录及其子目录的文件
- 如果文件在别的地方,需要提供完整路径
- Claude不会删除文件,除非你明确要求

## 第五部分：Case 1 - TikTok 直播+短视频运营分析（3500字，14分钟）

### 5.1 案例背景

**场景设定**：
你是一名直播+短视频运营,负责公司的TikTok账号。过去两个月发布了50场直播和短视频,现在需要分析数据,找出高转化内容的规律,为下个月的运营策略提供依据。

**手上的数据**：
一份完整的运营数据表格,包含50场直播和短视频的数据：
- **基础数据**：视频ID、标题、发布时间、播放量、点赞量、评论量、分享量、视频时长、视频类型、成单量
- **直播数据**：场观(UV)、在线峰值、人均停留时长、GMV、下单转化率
- **投流数据**：广告花费、CPM、ROI
- **互动数据**：弹幕数、礼物数、关注转化率

**老板的需求**：
1. 哪个时间段发布效果最好？
2. 哪类内容转化率最高？
3. 直播间表现如何？
4. 投流ROI怎么样？
5. 生成一份深度分析报告和可视化网页,下周例会要用

**学习目标**：
- 掌握提示词编写技巧
- 学会使用 `/live-ops-analytics` skill 生成深度分析报告
- 学会使用 `/frontend-design` skill 生成可视化网页
- 一个提示词完成所有分析

### 5.2 准备工作

**Step 1：获取数据文件**

本教程提供了完整的示例数据。你可以：

**Step 2：启动 Claude Code**

### 5.3 一个提示词完成所有分析

这是本教程的核心部分。你将学会如何编写一个完整的提示词,让 Claude Code 自动完成所有分析工作,包括生成深度分析报告和可视化网页。

**提示词编写原则**：
1. **清晰**：明确说明你想要什么
2. **具体**：详细描述每个部分的要求
3. **完整**：一次性说明所有需求,避免多次对话

**完整提示词示例**：

```
我有一份直播和短视频运营数据（tiktok_data.csv），包含50场直播和短视频的完整数据。

请使用 /live-ops-analytics skill 生成一份深度分析报告，包含：

1. 数据概览和关键指标
   - 总播放量、总GMV、平均转化率、平均ROI
   - 数据时间范围和样本量

2. 直播间表现分析
   - 场观(UV)分布和趋势
   - 在线峰值和人均停留时长
   - 下单转化率分析
   - 识别高转化直播的特征

3. 短视频效果分析
   - 播放量、互动率(点赞率、评论率、分享率)
   - 不同视频类型的表现对比
   - 发布时间对效果的影响
   - 视频时长与完播率的关系

4. ROI和投流分析
   - 广告花费分布
   - CPM(千次观看成本)分析
   - ROI排名和优化建议
   - 不同内容类型的投放效率

5. 用户互动分析
   - 弹幕数、礼物数分布
   - 关注转化率分析
   - 互动质量与转化的关系
   - 高价值用户特征

6. 优化建议和下月策略
   - 内容策略(类型、时长、发布时间)
   - 投流策略(预算分配、出价优化)
   - 转化优化(引导话术、优惠设计)
   - 具体执行计划和预期效果

报告要求：
- 适合给老板看的管理层汇报风格
- 每个建议都要有数据支撑
- 建议要具体可执行
- 包含预期效果预测

然后使用 /frontend-design skill 生成一个现代化的可视化网页，包含：

1. 顶部核心指标卡片（6-8个关键指标）
   - 总播放量、总GMV、平均转化率
   - 平均ROI、平均CPM、关注转化率
   - 使用卡片式设计,数字突出显示

2. 多维度数据图表
   - 发布时间vs播放量柱状图
   - 视频类型转化率对比图
   - ROI分布散点图
   - 转化漏斗图
   - 内容表现趋势折线图
   - 投流效率对比图

3. 数据表格
   - 高ROI内容Top 10
   - 待优化内容列表
   - 支持排序和筛选

4. 响应式布局
   - 适配桌面和移动端
   - 图表支持交互(悬停显示详细数据)

设计要求：
- 现代简洁风格
- 蓝白配色(主色#2563eb,辅色#60a5fa)
- 高质量的视觉呈现
- 专业的数据可视化

文件名：
- 报告：analysis_report.md
- 网页：dashboard.html
```

**复制上面的提示词,粘贴到 Claude Code 中,按回车发送。**

### 5.4 理解提示词的结构

让我们分析一下这个提示词为什么有效：

**第一部分：说明数据来源**
```
我有一份直播和短视频运营数据（tiktok_data.csv），包含50场直播和短视频的完整数据。
```
- 清楚地告诉 Claude 数据文件的位置和内容
- 让 Claude 知道数据的规模和类型

**第二部分：调用 skill 并说明需求**
```
请使用 /live-ops-analytics skill 生成一份深度分析报告，包含：
```

## 附录C：Mock数据设计方案

### Case 1：TikTok数据设计

**数据规模**：50条记录

**字段设计**：
```csv
video_id,title,publish_date,publish_hour,views,likes,comments,shares,duration,video_type,conversions
V001,3分钟学会做红烧肉,2026-01-05,19,28500,3420,285,570,180,教程,35
V002,今天又是打工人的一天,2026-01-06,12,12300,1476,123,246,45,剧情,8
...
```

**数据规律**：
1. **发布时间影响**：
   - 19:00-21:00发布的视频,播放量在20000-50000
   - 其他时段,播放量在5000-20000
   
2. **视频类型影响**：
   - 教程类：转化率6-10‱
   - 测评类：转化率4-8‱
   - 好物推荐：转化率3-7‱
   - 剧情类：转化率1-5‱

3. **互动数据关系**：
   - 点赞量 = 播放量 × (5%-15%)
   - 评论量 = 播放量 × (1%-3%)
   - 分享量 = 播放量 × (0.5%-2%)

4. **转化规律**：
   - 基础转化 = 播放量 × 转化率
   - 加入随机波动（±30%）
   - 3-5个异常值（高播放低转化）

**标题示例**：
- 教程类："3分钟学会XX"、"新手必看XX教程"、"手把手教你XX"
- 测评类："XX真的值得买吗"、"XX深度测评"、"XX使用一个月后"
- 好物推荐："这个XX太好用了"、"XX好物分享"、"XX必买清单"
- 剧情类："今天又是XX的一天"、"XX的日常"、"XX搞笑瞬间"
