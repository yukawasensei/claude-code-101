# 进阶技巧

本文档提供 Claude Code 的进阶使用技巧,帮助你提升工作效率和使用体验。

## YOLO模式 - 跳过权限确认

### 什么是YOLO模式?

YOLO模式(You Only Live Once)允许 Claude Code 自动执行所有操作,不再弹出权限确认提示。

### 如何启用?

```bash
claude --dangerously-skip-permissions
```

### 适用场景

- 处理大量重复性任务
- 你已经明确知道要做什么
- 在测试环境中快速迭代
- 完全信任 Claude 的操作

### 注意事项

⚠️ **使用此模式时要格外小心**:
- Claude 可能会执行删除、修改等操作
- 建议在测试环境或备份后使用
- 重要项目慎用

### 最佳实践

```bash
# 1. 先在测试环境验证
cd test-project
claude --dangerously-skip-permissions

# 2. 确认无误后再在正式环境使用
cd production-project
claude --dangerously-skip-permissions
```

## Plan 计划模式 - 先规划再执行

### 什么是Plan模式?

Plan 模式让 Claude 先制定详细的执行计划,等你确认后再执行,避免走弯路。

### 如何使用?

```bash
# 启动 Claude Code 后,按 Shift+Tab 切换到 Plan 模式
```

### 工作流程

1. **描述任务**: 告诉 Claude 你想做什么
2. **Claude 制定计划**: Claude 会列出详细的执行步骤
3. **你审核计划**: 检查计划是否合理
4. **确认执行**: 确认后 Claude 开始执行
5. **按计划推进**: Claude 按照计划逐步完成任务

### 适用场景

- 大型重构或架构调整
- 不确定最佳方案的任务
- 需要评估多个方案的场景
- 复杂的多步骤任务

### 示例对话

```
你: 我想重构这个项目的数据库层,使用 TypeORM 替换现有的 SQL 查询

Claude (Plan模式):
我制定了以下计划:
1. 分析现有数据库查询代码
2. 设计 TypeORM 实体模型
3. 创建迁移脚本
4. 逐个替换查询函数
5. 编写单元测试
6. 更新文档

是否执行此计划?

你: 确认

Claude: 开始执行...
```

## 上下文管理技巧

### 为什么要管理上下文?

- 对话太长会影响 Claude 的理解
- 上下文过多会降低响应速度
- 不相关的历史信息会干扰当前任务

### 何时清理上下文?

使用 `/clear` 命令清理上下文:

```bash
# 在 Claude Code 对话中
/clear
```

**最佳时机**:
- 完成一个独立任务后
- 开始新的分析任务前
- 感觉 Claude 理解出现偏差时
- 对话超过20轮后

### 上下文管理最佳实践

**1. 任务分段**
```
完成任务A → /clear → 开始任务B
生成报告 → /clear → 开始新分析
```

**2. 保存重要信息**

清理前,将重要信息保存到文件或 CLAUDE.md:

```
你: 请将刚才的分析结论保存到 findings.md
Claude: [保存文件]
你: /clear
```

**3. 定期清理**

建立清理习惯:
- 每完成一个阶段性任务就清理
- 每天开始工作前清理
- 切换项目时清理

## CLAUDE.md 使用详解

### 什么是 CLAUDE.md?

CLAUDE.md 是 Claude Code 的配置文件,会在每次对话时自动读取。你可以在这个文件中记录:
- 项目规范和编码风格
- 重要的技术决策
- 注意事项和约束条件
- 常用的提示词模板

### 如何创建?

```bash
# 在项目根目录创建
touch CLAUDE.md

# 或使用 Claude Code 创建
claude
你: 请创建一个 CLAUDE.md 文件,包含项目的基本规范
```

### CLAUDE.md 示例内容

```markdown
# 项目规范

## 编码规范
- 使用 TypeScript
- 组件使用函数式写法
- 样式使用 Tailwind CSS
- 所有函数必须有类型注解

## 数据分析规范
- 所有图表使用 Chart.js
- 配色方案:蓝色系 (#2563eb)
- 报告格式:Markdown
- 数据文件统一放在 data/ 目录

## 文件组织
- 源代码: src/
- 测试文件: tests/
- 文档: docs/
- 数据文件: data/
- 生成的报告: reports/

## 注意事项
- 不要修改 legacy/ 目录下的文件
- 所有数据库操作必须使用事务
- API 调用必须有错误处理
- 提交前必须运行测试

## 常用命令
```bash
npm run dev      # 启动开发服务器
npm run test     # 运行测试
npm run build    # 构建生产版本
```

## 提示词模板

### 数据分析
读取 data/[文件名],分析[维度],生成报告保存到 reports/

### 生成组件
创建 React 组件 [名称],使用 Tailwind CSS,函数式写法
```

### 使用技巧

**1. 初始化项目时创建**

新项目开始时就创建 CLAUDE.md,记录关键决策:

```markdown
# 项目初始化

## 技术栈选择
- 前端: React + TypeScript + Tailwind
- 后端: Node.js + Express
- 数据库: PostgreSQL
- 原因: [记录选择理由]

## 架构设计
[记录架构图和说明]
```

**2. 持续更新**

随着项目推进,不断更新 CLAUDE.md:

```markdown
## 2026-03-05 更新
- 新增用户认证模块
- 使用 JWT 进行身份验证
- Token 有效期: 7天
```

**3. 团队协作**

将 CLAUDE.md 提交到 Git,团队成员共享:

```bash
git add CLAUDE.md
git commit -m "Add project guidelines"
git push
```

## 图片分析功能

### 支持的模型

如果使用支持视觉能力的模型(如 Claude Opus、Sonnet),可以直接分析图片。

### 如何使用?

**方法1: 直接粘贴图片**

1. 复制图片(截图或文件)
2. 在 Claude Code 对话中按 `Command + V` 粘贴
3. 添加你的问题或要求

**方法2: 提供图片路径**

```
请分析这张图片: /path/to/image.png
[你的问题或要求]
```

### 使用场景

**1. 分析数据可视化图表**

```
你: [粘贴一张数据图表截图]
请分析这个图表,提取关键数据和趋势

Claude: 这是一个销售趋势图,显示...
```

**2. 根据设计稿生成代码**

```
你: [粘贴一张网页设计稿]
请根据这个设计稿,用 HTML + Tailwind CSS 实现

Claude: 我会创建一个匹配这个设计的网页...
```

**3. 识别图片中的文字和表格**

```
你: [粘贴一张包含表格的截图]
请提取这个表格的数据,保存为 CSV 文件

Claude: 我识别到以下数据...
```

**4. 分析错误截图**

```
你: [粘贴一张错误信息截图]
这个错误怎么解决?

Claude: 这是一个 [错误类型],原因是...
```

### 最佳实践

- 图片要清晰,避免模糊
- 截图时包含必要的上下文
- 配合文字说明,提高准确度
- 复杂图片可以分步分析

## 安全注意事项

### 数据安全

❌ **不要做**:
- 上传包含真实用户隐私的数据
- 上传公司机密文件
- 在提示词中包含敏感信息

✅ **应该做**:
- 使用脱敏数据或模拟数据练习
- 敏感字段用占位符替换
- 重要数据先备份

### API密钥安全

❌ **不要做**:
- 在代码或文档中写入 API 密钥
- 分享你的密钥给他人
- 将密钥提交到 Git

✅ **应该做**:
- 使用环境变量存储密钥
- 定期更换密钥
- 如果泄露,立即撤销

### 代码审查

- Claude 生成的代码要 review 再使用
- 特别注意文件删除、数据修改等操作
- 不确定的代码先在测试环境运行
- 重要操作前先备份

### 版本控制

- 重要文件要备份
- 使用 Git 管理项目文件
- 定期提交代码
- 定期导出分析结果

## 常见问题速查

### 问题1: Claude 说找不到文件

**原因**:
- 文件不存在
- 文件名拼写错误(注意大小写)
- 路径不正确

**解决方案**:
```bash
# 检查文件是否存在
ls -la

# 搜索文件
ls | grep data

# 使用绝对路径
请读取 /Users/你的用户名/Desktop/project/data.csv
```

### 问题2: 分析结果不符合预期

**原因**:
- 提示词不够具体
- 缺少上下文信息
- Claude 理解有偏差

**解决方案**:
```
请调整分析,我期望看到按月份分组的统计,而不是按天。
具体要求:
1. 按月份分组
2. 计算每月总和
3. 生成对比表格
```

### 问题3: 生成的图表样式不满意

**解决方案**:
```
请调整图表样式:
- 标题字号改为 24px
- 颜色改为 #2563eb
- 图例位置移到右侧
- 增加网格线
- 数据点标记改为圆形
```

### 问题4: 处理大文件很慢

**解决方案**:
```
请先分析文件大小,如果超过10MB:
1. 只读取前1000行进行采样分析
2. 或者分批处理
3. 或者只读取需要的列
```

### 问题5: 需要处理多个相似文件

**解决方案**:
```
我有10个CSV文件(data1.csv到data10.csv),结构相同。
请写一个脚本批量处理:
1. 读取所有文件
2. 合并数据
3. 统计总计
4. 生成汇总报告
```
